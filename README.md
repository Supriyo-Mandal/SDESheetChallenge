## 👋 About This Repository

This repository is my public journal for the **45-Day #SDESheetChallenge** by takeUforward.

Like many students preparing for software engineering interviews, I've solved problems before.

But this challenge is different.

Instead of collecting accepted submissions, I wanted to document the entire journey:

* What my first instinct was
* Where I got stuck
* Why the brute-force solution wasn't enough
* What observation changed my thinking
* How I eventually reached the optimal solution

The goal isn't to showcase perfect solutions.

The goal is to build a stronger problem-solving mindset — one day at a time.

## 🧠 My Problem Solving Workflow

For every challenge problem, I follow the same process:

```text
Read Problem
      ↓
Understand Constraints
      ↓
Build Brute Force
      ↓
Identify Bottlenecks
      ↓
Find Patterns
      ↓
Optimize
      ↓
Analyze Complexity
      ↓
Document Learnings
```

Sometimes the optimal solution comes quickly.

Sometimes it doesn't.

Both are part of the journey.
